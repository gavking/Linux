Pipes

Using |, you can send the output or input of commands

ls | more Passes output of ls to more command

Redirects

Using < and >, you can redirect the output or input of commands

ls > output.txt Passes output of ls into file output.txt
cat < input.txt Passes contents of input.txt into cat
cat < input.txt > output.txt Combines both of the above
cat > input.txt Allows you to write contents of file from terminal

Appends

Using << and >>, you can append to the output or input of commands

ls >> output.txt

STDOUT STDIN STDERR

STDIN (0) is the output stream to the terminal, STDOUT (1) is the input stream to the terminal, STDERR (2) is the Error stream They are referred to by number 0,1 and 2 respectively

ls -R / 2>errors.txt 1>output.txt Recursively list items from Root and send any errors to errors.txt, and other otput to output.txt

Can be combined with Pipes
